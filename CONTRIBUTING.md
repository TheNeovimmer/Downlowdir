# Contributing to Downlowdir

Thanks for your interest in contributing. Here's how to help.

---

## Reporting Issues

If something doesn't work, start by checking if there's already an issue about it. If not, open one.

When you report a problem, include:
- What you were trying to do
- What happened instead
- The exact command you ran
- Any error messages you saw

The more detail, the easier it is to reproduce and fix.

---

## Pull Requests

Before you submit a PR:

1. Fork the repository
2. Create a branch for your feature or fix
3. Make your changes
4. Run `npm run build` to make sure everything compiles
5. Test your changes if possible
6. Submit the PR

Keep your changes focused. If you find something unrelated while working on your PR, save it for a separate PR.

---

## Code Style

This project uses TypeScript. Keep your code consistent with what's already there:

- Use meaningful variable names
- Add types where obvious
- Keep functions small and focused
- Comment the tricky parts, not the obvious ones

---

## What Needs Work

Areas where help is most needed:

- More video site support
- Better error handling and recovery
- Performance improvements for chunking
- Cross-platform testing (especially Windows)

---

## Questions

For questions that aren't bug reports or feature requests, you can open a discussion on GitHub. But honestly, opening an issue with a question is fine too - it helps others who have the same question later.
